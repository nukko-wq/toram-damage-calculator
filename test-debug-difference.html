ダメージ差分計算デバッグテスト

問題の概要

実際の差分: 1,733,894 - 1,545,621 = 188,273

表示されている差分: +1,209

差異: 188,273 - 1,209 = 187,064

ドン・プロフンドクリスタルの効果

  • ATK_Rate: 10（攻撃力+10%）
  • STR_Rate: 7（STR+7%）
  • Critical_Rate: 8（クリティカル率+8%）
  • DEF_Rate: -27（DEF-27%）

デバッグ手順

  1. 開発者ツールのコンソールを開く（F12）
  2. アプリケーションでクリスタル選択モーダルを開く
  3. ドン・プロフンドクリスタルにマウスオーバーして差分表示を確認
  4. コンソールで以下のログを確認:
    • 🔧 CRYSTAL SIMULATION START
    • 🔧 CRYSTAL SIMULATION EXECUTED
    • ⚙️ CALCULATE RESULTS COMPARISON
    • 🎯 DAMAGE CALCULATION COMPARISON
    • 💡 calculateDamageFromResults called with
    • 🎯 Correct Damage Difference Calculation

調査すべき点

  1. クリスタルシミュレーション: simulateItemEquipSimpleが正しくクリスタルを装着しているか？
  2. 計算結果の更新: calculateResultsが新しいクリスタル効果を正しく反映しているか？
  3. ダメージ計算: calculateDamageFromResultsの入力と出力が正しいか？
  4. 数値の精度: 計算過程で数値の丸めや型変換による損失はないか？
  5. 設定の同期: DamagePreviewとuseDamageDifferenceCorrectで同じ設定を使用しているか？
